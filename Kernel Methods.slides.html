Primal v. Dual Representation

Recall the perceptron algorithm:

  • start with \(\theta = [0, 0]\)
  • While \(\theta\) is wrong
    • for each input vector
      • if \(\theta\) is wrong for this vector, add this vector to \(\theta\)

Lets make this a little more formal, using our classifying function from before:

  1. \(\theta = [0, 0]\)
  2. for \(i \in \{1, ..., N\}\)
    1. if \(y_i(\theta \cdot x_i) <= 0\)
      1. \(\theta = \theta + y_i x_i\)
  3. Repeat step 2 until no mistakes are made

Then to classify a new point \(x\), we use \(\mathrm{sign}(\theta \cdot x)\)

We mentioned before (but did not substantiate) that sometimes it's useful to define your classifier in terms of training points, rather than explicitly as a vector.

How can we re-write the perceptron algorithm this way?

1. Initialize \(\alpha_i = 0\), \(i \in \{1, ..., N\}\)

2. for \(i \in \{1, ..., N\}\)

  • if \(y_i(\theta \cdot x_i) <= 0\)
    • \(\alpha_i = \alpha_i + y_i\)

3. Repeat step 2 until no mistakes are made

So after this process, \(\alpha_i\) is a count of how many mistakes were made on element \(i\). Comparing to the previous formulation, it is also a count of how many times that element \(i\) was added to \(\theta\).

How do we classify using \(\alpha\)?

One option is to set \(\theta = X \alpha\) and classify with the same classification function before. This defines \(\alpha\) as a vector of weights. When you sum up the training data, each weighted by the corresponding \(\alpha\) coefficient, you get \(\theta\)

But then we're back where we started.

Notice that if \(\theta = X \alpha\), and our classifier is \(\mathrm{sign}(\theta \cdot x)\), we can redefine our classifier in terms of \(\alpha\) directly.

\[ y = \mathrm{sign}(X \alpha \cdot x) \]

Expanding out the matrix multiplication, this becomes:

\[ y = \mathrm{sign}\left(\sum_{i \in s.v.} \alpha_i x_i \cdot x \right) \]

where \(i \in s.v.\) means the set of \(i\) that index the support vectors.

Notice that we can now train the classifier and then classify new data, all without explicity using \(\theta\). This is known as the dual form of the perceptron.

Notice:

  • Ideally most of the \(\alpha\) coefficients are zero, which means that the corresponding values can be removed from \(X\) after training
  • Evaluating this directly would be the same as before

The above plot is the same data but with a transformation applied that adds a 2nd dimension. The 2nd axis is simply the square of the given data. See how now we can clearly find a linear separator in the data. Also note that our ability to find a linear separator is not dependent on a coefficient on the squared term.

Let's go ahead and give a name to this new space, how about \(\phi(x)\). Here's the procedure:

  1. create a new feature matrix \(\Phi\) from our given data \(X\), where each vector \(\phi_i = \phi(x_i)\).
  2. train your classifier on vectors in the new \(\Phi\) space, giving you \(\alpha\). Remember(for later) that in the training process for the percepteron the training happens at each step by applying dot products between the current classifier and the training points (in \(\Phi\) space).
  3. a new data \(x\) is classified by applying the same classifier as before, but first transform into the \(\Phi\) space, so

\[ y = \mathrm{sign}\left(\sum_{i \in s.v.} \alpha_i \phi(x_i) \cdot \phi(x) \right) \]

Again, you might be tempted at this point to say "So what? We could still just be using \(\theta\) instead of all this \(\alpha\) stuff, and indeed are effectively using it in the next step when we evaluate \(\alpha_i \phi(x_i) \cdot \phi(x)\)."

The Kernel Trick

Here's where things get fancy. There are a set of transforms for which you can calculate \(\phi(x) \cdot \phi(x')\) directly given \(x\) and \(x'\), without actually going into the feature space and calculating the dot product directly. This substitution is known as the "Kernel Trick". In some cases there are good approximations to evaluating the kernel, providing additional efficiency. This is why it is important in the dual formulation that we can express all the interactions as dot products in the feature space, and thus apply the kernel. Often you see it listed as \(K(x, x')\), for example the quadratic kernel we used in the example above can be described with:

\[ \phi(x) \cdot \phi(x') = K(x, x') = \left(\sum_{i=1}^n x_i x_i' + 1\right)^2 \]

Notice that the sum is equivilent to a dot product in the original space, which is (possibly significantly) lower dimensionality than the feature space.

The RBF Kernel

One kernel that's especially useful is the "Radial Basis Function" or RBF. One way to think of the RBF is that it is a measure of distance. That is, \(K_{\mathrm{RBF}}(x, x')\) is a measure of how far apart \(x\) and \(x'\) are. More specifically:

\[ K(x, x') = \exp\left(\gamma ||x - x'||^2\right) \]

which can be thought of as a similarity measure with a gaussian roll-off. You'll sometimes see \(\gamma\) as \(-\frac{1}{2 \sigma}\), to more closely follow the standard gaussian formula.

Notice that if \(x = x'\), \(K(x, x') = 1\) and as \(||x - x'||. \rightarrow \infty\), \(K(x, x')\) approaches zero.

An interesting thing to note is that we would have a hard time computing \(\theta\) directly in this case, as the feature space implied by the RBF is infinite-dimensional.

# Utility Code


import matplotlib.pylab as pl
import numpy as np
from sklearn import svm

def plot_sep(X, Y, theta, ax=None, theta_label=True):
    # get the decision boundary
    a = -theta[0] / theta[1]
    sep_x = np.linspace(-5, 5)
    sep_y = a * sep_x

    # draw it
    if ax is None:
        fig = pl.figure(figsize=(8, 8))
        ax = fig.add_subplot(1, 1, 1, xlim=[-5,5], ylim=[-5,5])
        ax.grid(True)
    ax.scatter(X[:, 0], X[:, 1], c=Y, cmap=pl.cm.Paired)
    ax.plot(sep_x, sep_y, "k--")
    ax.quiver([0], [0], theta[0], theta[1], angles='xy', scale_units='xy', scale=1)
    if theta_label:
        ax.text(theta[0], theta[1], r"$\theta$", fontsize=24)

def plot_1d(X, Y):
    pl.scatter(X, np.zeros(len(X)), c=Y, cmap=pl.cm.Paired, s=100)
    ax = pl.gca()
    ax.yaxis.set_visible(False)
    ax.spines["bottom"].set_position('zero')
    ax.spines["bottom"].set_smart_bounds(True)
    for direction in ["right","left", "top"]:
        ax.spines[direction].set_color('none')
    ax.xaxis.set_ticks_position('bottom')
    
def demo_rbf(N, C=10, gamma=0.1):
    np.random.seed(0)
    # note that np.r_ concatenates the given arrays along their first axis
    X = np.r_[np.random.randn(N/2, 2), np.random.randn(N/4, 2) - [1.5, 1.5],  np.random.randn(N/4, 2) + [1.5, 1.5]]
    Y = np.r_[np.zeros(N/2), np.ones(N/2)]

    # fit the model
    clf = svm.SVC(kernel='rbf', C=C)
    clf.fit(X, Y)

    print("%d support vectors" % len(clf.support_))
    return X, Y, clf

def plot_rbf(X, Y, classifier, ax=None):
    if ax is None:
        fig = pl.figure(figsize=(8, 8))
        ax = fig.add_subplot(1, 1, 1, xlim=[-5,5], ylim=[-5,5])
        ax.grid(True)
    meshsize = 100
    xx, yy = np.meshgrid(np.linspace(-5, 5, meshsize), np.linspace(-5, 5, meshsize))
    Z = classifier.decision_function(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    pl.contour(xx, yy, Z, 3, colors=['blue', 'black', 'red'], linestyles='dashed')

    ax.scatter(classifier.support_vectors_[:, 0], classifier.support_vectors_[:, 1],
           s=80, facecolors='none')
    ax.scatter(X[:, 0], X[:, 1], c=Y, cmap=pl.cm.Paired)
