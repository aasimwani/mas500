The Perceptron Algorithm¶

Classification

  • Focus on binary classification
  • Assume we're given some known "training data" with labels (-1 and 1) for each point

Some Notation

let \(X = \{x_1, x_2, \ldots, x_n\}\) be our training data

For these examples \(x_i \in R^2\), so it's easier to plot. The math works for higher-dimensional data as well.

\(Y = \{y_1, y_2, \ldots, y_n\}\) is the set of labels, so \(y_i \in \{-1, 1\}\)

The goal is to find a vector \(\theta\) so that for some new data point \(x\) in class \(y\), \(\mathrm{sign}(\theta \cdot x) = y\). This can also be expressed as \(y (\theta \cdot x) > 0\)

The below plot shows random points classified by a decision boundary (dashed black line), which is defined by the vector \(\theta\)

The Perceptron Algorithm

How do we find \(\theta\)?

start with \(\theta = [0, 0]\)

  • While \(\theta\) is wrong
    • for each input vector
      • if \(\theta\) is wrong for this vector, add this vector to \(\theta\)

NB:

  • If the data is not linearly separable, then the algorithm will go on forever
  • As given here, the separator must go through the origin
  • We don't care about the magnitude of \(\theta\) here, only its direction

Given a set of data and labels, how does one find an appropriate \(\theta\) vector? The Perceptron algorithm is one of the earliest algorithms to come up with one, assuming that the data is in fact linearly separable.

We start with an initial value \(\theta = [0, 0]\). For each training data point we try our current \(\theta\) to see if it works for that data point. If it does then we continue to the next point. If it is wrong, we update \(\theta\) by adding to it the vector representing the test point. If we make it all the way through the data without needing to do an update, than our classifier correctly classifies all the training data and we're done.

There are a few important things to notice:

  1. If the data is not linearly separable, then the algorithm will go on forever. If the data is linearly separable, this algorithm is proven to terminate eventually. We won't be going into the proof here, but the idea is that it number of mistakes it will make is inversely proportional to the margin in the data
  2. For simplicity we constrain the classifier to go through the origin.
  3. We don't care about the magnitude of \(\theta\) here, only its direction

There are many variants of the perception algorithm that can resolve these issues, but this is the simplest.

def perceptron_demo(N, max_mistakes=1000):
    X = np.r_[(np.random.randn(N/2, 2) + [-2, -2]), (np.random.randn(N/2, 2) + [2, 2])]
    Y = np.array([-1] * (N/2) + [1] * (N/2))

    theta = np.array([0, 0])
    dirty = True
    mistakes = 0

    while dirty and mistakes < max_mistakes:
        dirty = False
        for i in range(N):
            if Y[i] * theta.dot(X[i]) <= 0:
                mistakes += 1
                dirty = True
                theta = theta + Y[i] * X[i]

    print("%s after %d mistake%s" % ("Aborted" if dirty else "Finished",
                                     mistakes,
                                     "" if mistakes == 1 else "s"))
    return X, Y, theta

Step by Step

Here we show step-by-step how the \(\theta\) vector updates as it makes mistakes, eventually converging on a solution. For this example we've given it a harder problem than above by uniformly filling the space with points. We ensure that they are linearly separably by definition, by setting the ground-truth using a linear classifier unknown to the algorithm.

At each step we also highlight the mis-classified point. Looking at the point in the previous frame you can see that each one is mis-classified.

import matplotlib.pylab as pl
import numpy as np

def plot_sep(X, Y, theta, ax=None, theta_label=True):
    # get the decision boundary
    a = -theta[0] / theta[1]
    sep_x = np.linspace(-5, 5)
    sep_y = a * sep_x

    # draw it
    if ax is None:
        fig = pl.figure(figsize=(8, 8))
        ax = fig.add_subplot(1, 1, 1, xlim=[-5,5], ylim=[-5,5])
        ax.grid(True)
    ax.scatter(X[:, 0], X[:, 1], c=Y, cmap=pl.cm.Paired)
    ax.plot(sep_x, sep_y, "k--")
    ax.quiver([0], [0], theta[0], theta[1], angles='xy', scale_units='xy', scale=1)
    if theta_label:
        ax.text(theta[0], theta[1], r"$\theta$", fontsize=24)
        

def plot_slide_linear_sep():
    fig = pl.figure(figsize=(8,4))

    np.random.seed(0) # ensure repeatable results

    sep1 = (np.random.randn(20, 2) + [-2, -2])
    sep2 = (np.random.randn(20, 2) + [2, 2])

    nosep1 = (np.random.randn(20, 2) + [-1, -1])
    nosep2 = (np.random.randn(20, 2) + [1, 1])

    ax1 = fig.add_subplot(1, 2, 1)
    ax1.set_title("Linearly Separable")
    ax1.scatter(sep1[:, 0], sep1[:, 1], c="red")
    ax1.scatter(sep2[:, 0], sep2[:, 1], c="blue")
    ax2 = fig.add_subplot(1, 2, 2)
    ax2.set_title("Not Linearly Separable")
    ax2.scatter(nosep1[:, 0], nosep1[:, 1], c="red")
    ax2.scatter(nosep2[:, 0], nosep2[:, 1], c="blue")

def plot_slide_step_by_step():
    N = 100
    true_theta = np.array([-1.0, 2.0])
    # generate uniform random points in a 10x10 box
    np.random.seed(0)
    X = (np.random.rand(N, 2) - [0.5, 0.5]) * 10
    Y = np.sign(X.dot(true_theta))

    theta = np.array([0, 0])
    dirty = True

    fig = pl.figure(figsize=(25, 13))
    j = 0

    while j < 16 and dirty:
        dirty = False
        for i in range(N):
            if j < 16 and Y[i] * theta.dot(X[i]) <= 0:
                theta = theta + Y[i] * X[i]
                ax = fig.add_subplot(3, 5, j+1, xlim=[-5,5], ylim=[-5,5])
                plot_sep(X, Y, theta, ax, False)
                ax.scatter(X[i, 0], X[i, 1], s=80, facecolors='none')
                dirty = True
                j += 1
